# Kodexempel

Kodexemplen visar ansvar, lager och designmonster for objektorienterad design.

Avgransningar:
- Inte ett komplett korbart system
- Ingen databas eller API-hosting
- Fokus pa domanlogik, use-case och abstrahering

Mappar:
- `Domain`: entiteter, value objects, domanhandelser
- `Application`: use-case tjanster och interfaces
- `Infrastructure`: tekniska implementationer

Nya exempel:
- `Application/QueryOrders/QueryOrdersService.cs` visar admin-flode med filtrering pa orderstatus
